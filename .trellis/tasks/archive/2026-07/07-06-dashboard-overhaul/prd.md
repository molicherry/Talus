# Beszel 风格仪表盘 + 服务器详情重构

## 目标

将 Talus 前端重构为 Beszel 风格的两级视图架构：仪表盘概览（首页）→ 服务器详情（一站式操作中心）。用户登录后立即看到所有服务器的健康状态，点击任意服务器进入详情页完成监控、终端、命令执行等操作。

参考设计：Beszel (henrygd/beszel) — 以卡片网格展示多服务器概览，颜色编码状态指示器，嵌入式图表详情页。

## 目标用户

单运维人员管理 5-15 台 Linux 服务器。需要"一眼看全"的健康概览 + "一站式"的服务器操作入口。

## 功能需求

### D1: 仪表盘首页

**位置**: `features/dashboard/`（新建模块）
**路由**: `/`（替换当前占位 `DashboardPage`）

**布局**: 响应式卡片网格 `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`

**服务器卡片内容**:

```
┌─────────────────────────────────┐
│ [🟢] 服务器名称           [ ⋮ ] │  ← 状态点 + 名称 + 操作菜单
│─────────────────────────────────│
│ 🖥 CPU    [████████    ]  12.5%  │  ← 进度条 + 百分比
│ 💾 Mem    [████        ]  45.2%  │
│ 💿 Disk   [██████████  ]  78.1%  │
└─────────────────────────────────┘
```

**状态点颜色**:
- 🟢 在线 (`bg-green-500`)，带 CSS `animate-ping` 脉冲环
- 🔴 离线 (`bg-red-500`)
- 🟡 检查中 (`bg-yellow-500`)
- ⚪ 未监控 (`bg-gray-400`)

**进度条颜色**: 绿(<65%) / 黄(65-90%) / 红(>90%) / 灰(无数据)

**交互**: 点击卡片 → `/servers/:id`，操作菜单 → 编辑/删除，60s 自动刷新

**状态处理**: 加载骨架 → 空状态引导 → 错误重试

### D2: 服务器详情页重构

**路由**: `/servers/:id`（整合监控图表、保留终端/命令执行独立路由入口）

**页面结构**:

```
┌─ 信息栏 ───────────────────────────────────────────┐
│ 服务器名称  [🟢 在线]  |  🐧 OS  |  ⏳ 运行时间      │
│ 🖥 CPU型号  |  🧠 内存总量   |                       │
│                    [编辑] [终端] [执行命令]          │
└─────────────────────────────────────────────────────┘
┌─ 监控图表（2 列网格 xl:grid-cols-2）────────────────┐
│ ┌── CPU ───┐ ┌── 内存 ──┐                         │
│ │ AreaChart │ │ AreaChart │                         │
│ └──────────┘ └──────────┘                         │
│ ┌── 磁盘 ───┐                                       │
│ │ BarChart  │                                       │
│ └──────────┘                                       │
│               [1h] [6h] [24h] [7d]                 │
└─────────────────────────────────────────────────────┘
```

- `/servers/:id/monitoring` → **重定向**到 `/servers/:id`
- `/servers/:id/terminal` → 保留独立路由，详情页添加入口按钮
- `/servers/:id/exec` → 保留独立路由，详情页添加入口按钮

### D3: 状态指示器系统

**复用组件**: `components/ui/status-indicator.tsx` + `components/ui/metric-bar.tsx`

| 状态 | 颜色类 | 脉冲 |
| online | `bg-green-500` | ✅ |
| offline | `bg-red-500` | ❌ |
| checking | `bg-yellow-500` | ✅ |
| unknown | `bg-gray-400` | ❌ |

### D4: 后端数据支撑

**方案 A（推荐）**: `GET /api/v1/servers` 响应增加 `status` 和 `latest_metrics` 字段
**方案 B**: 新增 `GET /api/v1/metrics/summary?server_ids=1,2,3`

> 与后端确认后选择方案。方案 A 改动最小。

## 非功能需求

- **性能**: 仪表盘首屏渲染 < 1s（15 台），图表 < 2s（7 天数据）
- **刷新**: 60s 自动，不阻塞 UI
- **响应式**: 桌面/平板/手机
- **可访问性**: 状态指示器附带 sr-only 文本
- **暗色模式**: 所有新组件支持 Tailwind `dark:` 变体
